# EIC repository execution boundary

This repository is maintained through AI-assisted engineering workflows.

Authoritative execution order:

1. Current explicit user instruction.
2. Nearest path-scoped `AGENTS.md`.
3. Root `AGENTS.md`.
4. `AI_CODING_DOCTRINE.md`.
5. Executable schemas, checks, tests, and build code.
6. README and historical reports.

The normalized registry built from `config/site.json` and every public `tools/*/tool.json` owns all public SEO, AI-discovery, route, claim, and freshness state. Generated output is disposable and must never be edited manually.

Deployment claims require evidence from the owner route. A GitHub commit, local build, preview deployment, and production custom-domain response are separate verification states.
