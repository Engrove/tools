# EIC repository execution boundary

This repository is maintained through AI-assisted engineering workflows.

Authoritative execution order:

1. Current explicit user instruction.
2. Nearest path-scoped `AGENTS.md` for local repository invariants.
3. Root `AI_CODING_DOCTRINE.md` for repository-wide AI Vibe coding methodology, evidence semantics, test status, readback, stop rules, and reporting.
4. Root `AGENTS.md` for repository architecture, production invariants, exact commands, and publishing rules.
5. Executable schemas, checks, tests, and build code.
6. README and historical reports.

A nearer path-scoped instruction may specialize repository behavior but must not weaken the doctrine's correctness, safety, evidence, verification, readback, or anti-overclaim requirements.

The normalized registry built from `config/site.json` and every public `tools/*/tool.json` owns all public SEO, AI-discovery, route, claim, and freshness state. Generated output is disposable and must never be edited manually.

Repository, test, runtime, deployment, file-content, and visual claims are separate evidence domains. Deployment claims require evidence from the owner route. A GitHub commit, local build, CI run, preview deployment, and production custom-domain response are distinct verification states.
